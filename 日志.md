### 1. 项目背景

### 3. 8-bit 头像系统（重点功能）
用户提出要替换照片 OOTD 为 8-bit 像素小人：
- 创建完整的像素头像构建器
- 7个分类：体型、发型、上衣、下装、鞋子、配饰（最多2个）、配色
- 实时预览功能
- "我的小人衣柜" - 保存最多12个头像
- 在所有页面展示像素头像

### 4. 遇到的问题和修复

#### Prisma 7 兼容性问题
- **问题**: Prisma 7 强制要求使用 adapter，导致初始化失败
- **解决**: 降级到 Prisma 6.19.3，恢复稳定运行

#### API 数据不匹配错误
- **问题**: `participants.find()` 报错 undefined is not an object
- **原因**: fetchData 调用了错误的 API endpoint
- **解决**: 改为调用 `/api/spaces/[code]/participants`

#### 缓存损坏问题
- **问题**: Next.js 缓存文件损坏导致 internal server error
- **解决**: 清理 `.next` 目录，重新构建

#### Google Fonts 网络错误
- **问题**: 构建时无法连接 Google Fonts
- **解决**: 移除 Geist 字体，改用系统字体

### 5. 手机访问方案

用户需要在手机上查看应用，讨论了两个方案：

#### 方案一：ngrok（最快）
- 优势：5分钟完成，公网可访问
- 需要：authtoken

#### 方案二：Vercel 部署（长期稳定）

## 技术决策

### 为什么选择 Vercel 而不是其他方案？

**考虑过的方案：**
1. **ngrok** - 最快但需要电脑开机
2. **本地 IP** - 最简单但需要同 WiFi
3. **Vercel** - 长期稳定，全球访问，自动 HTTPS

**选择 Vercel 的原因：**
- 用户想要稳定的公网访问
- Next.js 官方推荐平台
- 免费额度足够使用
- 自动部署和 CDN

### 数据库迁移策略

**从 SQLite 迁移到 PostgreSQL：**
- 更新 Prisma Schema
- 安装 PostgreSQL 依赖
- 配置环境变量
- 创建迁移文件
